Claude Codeのエコシステムにおいて、GitHubで驚異的な137,000以上のStarsを獲得し、Anthropic公式ハッカソンで受賞した、現在最も強力なClaude Code強化ツールとされるプロジェクトがあります。それが `everything-claude-code` です。

今日は、このプロジェクトの全容を解き明かし、Claude Codeにもたらすものと、素早く使い始める方法を紹介します。

## これは何か

`everything-claude-code` は単なる設定パックではなく、完整的なAIエージェントパフォーマンス最適化システムです。スキル（Skills）、本能（Instincts）、メモリ（Memory）、セキュリティスキャン、研究優先開発などのモジュールをClaude Codeに重ね合わせ、任务完了効率を数倍に引き上げることを約束します。

対応AIエージェントツール：**Claude Code、Codex、Cowork** など。

## 核心機能

### トークン最適化

システムにはトークン消費監視とインテリジェント圧縮戦略が組み込まれています。各セッションでトークン使用量を追跡し、しきい値に達すると自動的にコンテキスト圧縮をトリガーし、コンテキストオーバーフローによる对话中断や繰り返しを防止します。

### セッション越えメモリ永続化

デフォルトでは、Claude Codeの各セッションは「白紙」状態です。`everything-claude-code`は永続化レイヤを提供し、以下を実現します：
- プロジェクトの、技術的負債とトゥドゥリストを記録
- ユーザー設定をセッション間で保持
- コードパターンとアーキテクチャ規約を自動抽出

### 専門サブエージェントオーケストレーション

38のサブエージェントがそれぞれの領域を担当：
- **プランナーエージェント**：复杂な任务を実行可能なステップに分解
- **アーキテクトエージェント**：コード構造をレビューし、改善を提案
- **コードレビュアー**：潜在的なバグとセキュリティ脆弱性を自動スキャン
- **リファクタリング専門家**：增量的なコード преобразование を実行
- **テストエンジニア**：覆盖率が高く冗長性のないテストケースを生成

複数のサブエージェントは並列に协作したりチェーン呼び出しを形成し、完全な任务パイプラインを構築できます。

### 検証ループ

任务実行後、システムは自動的に検証ループを実行します：
1. ユニットテスト
2. インテグレーションテスト（オプション）
3. コード品質スキャン
4. ドキュメント整合性チェック

全検証ステージをパスした場合のみ、結果がコミットされます。

### モデルルーティング

組み込みの `/model-route` コマンドは、任务の複雑さに基づいて自動的に最適なモデルを選択し続けます：
- 简单なクエリ → 軽量高速モデル（低コスト）
- 複雑な推論 → 强力推論モデル（高品質）

## インストールと設定

### ステップ1：プラグインをインストール

```bash
# プラグインマーケットプレイスを追加
/plugin marketplace add affaan-m/everything-claude-code

# プラグインをインストール
/plugin install everything-claude-code@everything-claude-code
```

### ステップ2：ルールセットをインストール

```bash
# リポジトリをクローン
git clone https://github.com/affaan-m/everything-claude-code.git
cd everything-claude-code

# 完整インストール
./install.sh --profile full

# または言語別にインストール（TypeScript と Python の例）
./install.sh typescript python
```

Windows ユーザーの場合：
```powershell
.\install.ps1
```

### ステップ3：使用開始

インストール完了後、Claude Code で以下の新コマンドを使用できます：

```bash
/everything-claude-code:plan          # プランニングフローをトリガー
/harness-audit                         # 現在のツール効率を監査
/loop-start                            # ループ実行を開始
/quality-gate                          # 品質ゲートチェックを実行
/model-route                           # インテリジェントモデルルーティング
```

## 精选コマンド

### `/harness-audit` — ツール使用監査

現在のセッションにおけるすべてのツール呼び出しの頻度、所要時間、成功率は оптимизация提案を分析します。遅い Agent のデバッグやトークンの浪费診断に非常に便利です。

### `/quality-gate` — 品質ゲート

コードコミット前に包括的な品質チェックリストを実行します：
- 構文チェック
- リント（ESLint / Pylint 等）
- ユニットテスト覆盖率
- セキュリティスキャン（OWASP Top 10）

### PM2 マルチエージェントオーケストレーション

PM2 でマルチ Agent 協調クラスターを構成し、大規模自动化任务を処理できます：

```yaml
# pm2.config.js 示例
module.exports = {
  apps: [{
    name: 'claude-planner',
    script: 'agents/planner.js',
    instances: 1,
    exec_mode: 'cluster'
  }, {
    name: 'claude-coder',
    script: 'agents/coder.js',
    instances: 'max',
    exec_mode: 'cluster'
  }]
}
```

## なぜ137k Starsなのか

GitHub の Claude Code 関連17,800以上のリポジトリの中で、`everything-claude-code` の成功はいくつかの要因に起因します：

**1. 比類のない網羅範囲**
単一機能の強化ではなく、エージェント、スキル、ルール、メモリにわたる完整システムです。ユーザーは直接生產性を向上できる「一括パック」を手に入れます。

**2. 高いカスタマイズ性**
156のスキルは需要的有効化でき、言語ルールセットは增量インストールに対応しています。ユーザーはプロジェクトの实际情况に応じて選択的に統合できます。

**3. コミュニティ駆動、持続的迭代**
Anthropic ハッカソン受賞後、継続的にコミュニティからの寄稿を受け付け、高いアクティブさを維持しています。

**4. オープンで透明**
すべてのルール、プロンプト、ワークフロー定義はオープンでレビューの対象です。ユーザーはすべての強化の背後にあるロジックを理解できます。

## 活用シーン

- **大規模コードベースメンテナンス**：マルチエージェント協調により大規模プロジェクトの迭代速度が大幅に向上
- **コードレビュー自動化**：CI/CDパイプラインに品質ゲートを組み込む
- **研究任务**：セッション越えメモリによりAIが复杂な研究线索を継続的に跟踪
- **チーム標準化**：統一されたルールセットをチーム内に配布し、コードスタイルの一貫性を確保

## 潜在的な制約

- **学習曲線がやや厳しい**：初期設定では多个モジュールの相互作用を理解する必要がある
- **リソース开销**：38のサブエージェントが並列実行的时候のメモリ消费は無視できない
- **ルールセット膨胀**：ルールが増えるに伴いメンテナンスコストが上昇する可能性

## まとめ

`everything-claude-code` の137k Starsは、Claude Code強化ツールへの大きなコミュニティ需要を证明了。それは単なるプロンプト最適化ではなく、マルチエージェントシステム、持続的メモリ、品質ゲート、モデルルーティングを融合した完整的エンジニアリングソリューションです。

Claude Codeでより強力な自动化パイプラインを構築したい、またはAIエージェントに真の意味で長期記憶の協調パートナーを持たせたい場合、このプロジェクトは深入りする価値があります。
